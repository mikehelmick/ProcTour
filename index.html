Welcome to ProcTour

ProcTour is a distributed algorithms simulator written in Java. Using a single JVM instance, ProcTour makes it easy to code and test distributed algorithms without worrying about things like discovery, bootstrapping, network programming, or serialization. You can write algorithms by extending the Proc class and responding to two types of messages (1) Heartbeats, via the tick method and (2) messages via the receiveMessage method. Processes can send messages in either unicast or multicast fashion. An example (and a poor example at that) is available here. Defining your own clock class is critical, an interface is provided for clocks, but it is not functional. Some kind of virtual clock implementation is recommended.

Getting Started

You will need git, java 1.6/1.7, and Apache buildr in order to acquire and build the source code.

$ git clone https://github.com/mikehelmick/ProcTour.git
  $ buildr package
  $ buildr install

The install command will install the necessary jar files in your local maven repository (buildr uses maven repositories).

To run the example program that comes with ProcTour, run the command:

$ buildr run

Validators

ProcTour includes a system for distributed algorithm validators. Currently, only a simple validator for distributed mutual exclusion is provided. This validator does not validate that your code actually implements distributed clocks, just that resources are only accessed by 1 process at a time, and ever release occurs after a corresponding acquire.

More validators are planned!
